Elderly Monitor — Design Spec

Review before implementation · 2026-04-20

Problem

Monitor an elderly person's wellbeing using devices already in the home. Detect abnormal inactivity and alert family — no expensive hardware, zero ongoing cost.

Architecture

Android (Termux) ──→ Google Sheets API
Alexa (IFTTT)    ──→ Google Sheets API
CCTV (kensho)   ──→ Google Sheets API
                     │
      Apps Script Rules Engine (every 10 min)
                     │
Telegram       + Sheets Dashboard + CallMeBot (emergency)

Data Sources — What Gets Tracked

SourceSignalsHow
📱 AndroidMovement, calls, notificationsTermux script every 5 min
🔊 AlexaTimer setIFTTT → Sheets webhook
📷 CCTVOutdoor motionkensho-local patch

Active hours: 4:00 AM – 10:00 PM IST (respects mom's early wake-up)

Rules (editable in Sheets — no code change needed)

RuleThresholdAlert Level
No phone movement3 hrs, 4am–10pm⚠️ Warning
No phone usage (calls/notif)2 hrs, 4am–10pm⚠️ Warning
No morning activity by 6amdaily check⚠️ Warning
Combined silence (all sources)2 hrs, 4am–10pm🔴 Alert → Phone call

1-hour cooldown per rule — no repeated alerts

Files to Build

elderly-monitor/
├── termux/monitor.sh        — runs on Android every 5 min
├── sheets/rules_engine.gs   — Apps Script rules checker
├── sheets/alert_sender.gs   — Telegram + CallMeBot dispatch
├── kensho-patch/cctv_sheet_hook.py — patches kensho-local
└── DEPLOY.md                — full setup guide

Success Criteria

  • Phone activity in EVENTS sheet within 5 minutes
  • Alexa + CCTV events within 1 minute
  • DASHBOARD shows live last-seen for all 3 sources
  • Warning fires on Telegram at threshold
  • CallMeBot WhatsApp call fires on combined silence
  • Zero ongoing cost — runs on Google free tier
